** RFC 2119 Usage

The key words =MUST=, =MUST NOT=, =SHOULD=, =SHOULD NOT=, and =MAY= in this
document and the role files are to be interpreted as normative instruction
terms.

** Compile / Export Gate

No section may be treated as compile/export eligible unless it:

- uses RFC 2119 normative language when making enforceable claims
- answers at least one invariant question
- references at least one algorithm lane
- declares authority, surface, and artifact role
- declares proof-burden and convergence metadata

Idea-mode sections MAY remain conversational, but they MUST NOT be treated as
implementation-relevant exports.

* 112 Proof Matrix

The framework uses:

- 8 invariant questions
- 7 algorithm lanes
- 2 proof directions for every lane

Therefore:

#+begin_example
8 x 7 x 2 = 112 proofs
#+end_example

This root file defines the matrix and links into the role files.
The role files remain conversational; the matrix gives them grounding.

* Semantic Adapter Boundary

Semantic adapters are downstream query, relation, and expression surfaces.
They are not runtime law.

This includes:

- S-expressions
- M-expressions
- Datalog
- Prolog
- recursive fixed-point discussion
- WordNet-style semantic lattice overlays
- the currently present package tree under [[file:org/packages/prologdb-5WN][org/packages/prologdb-5WN]]

Normative rules:

- Semantic adapters MAY query and relate canonical exports.
- Semantic adapters MUST NOT define canonical truth.
- Semantic adapters MUST NOT define address existence.
- Semantic adapters MUST NOT define closure.
- WordNet-style overlays MAY support naming, subsumption, and neighborhood queries.
- WordNet-style overlays MUST NOT replace replay-derived structure.
